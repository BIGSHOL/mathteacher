### 1단원: 덧셈과 뺄셈 - 자릿값의 이해와 재구조화

> 세 자리 수의 덧셈과 뺄셈으로 자연수 연산을 완성하는 단원. 받아올림과 받아내림의 원리 이해가 핵심.

#### 세부 개념 01: 받아올림 있는 덧셈 (`concept-e3-add-sub-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **받아올림**: 각 자리의 합이 10 이상일 때 상위 자릿수로 1 올림<br>- **자릿값 개념**: 같은 숫자라도 위치에 따라 나타내는 값이 다름<br>- 세로셈에서 자릿수 정렬의 중요성<br>- 계산 결과의 어림 (수 감각) |
| **주요 오개념** | - **받아올림 누락**: 십의 자리 합이 12인데 백의 자리로 1 안 올림<br>- **자릿수 정렬 오류**: 가로셈→세로셈 변환 시 왼쪽 정렬<br>- 받아올림한 1의 의미 이해 부족 (10을 의미) |
| **문항 설계 지침** | ✓ 받아올림의 필요성 인식 문항<br>✓ 중간 계산 과정 빈칸으로 받아올림 표기 강제<br>✓ 자릿수 정렬 연습 문항<br>✓ 어림으로 결과 범위 예측 |

#### 세부 개념 02: 받아내림 있는 뺄셈 (`concept-e3-add-sub-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **받아내림(재구조화)**: 상위 자리에서 1을 빌려 10으로 바꾸어 계산<br>- **핵심**: 빌려온 1 = 10 (자릿값의 이해)<br>- 세로셈에서 각 자리별 뺄셈 처리<br>- 검산으로 결과 확인 |
| **주요 오개념** | - **감산 방향성 오류** (가장 빈출): $423-157$에서 일의 자리 $3-7$이 안 되니까 $7-3=4$로 계산<br>  (원인: '큰 수-작은 수' 규칙을 위치 무관하게 적용)<br>- **연속 받아내림 오류**: 십의 자리가 0일 때 백의 자리에서 빌려오기 실패<br>- 받아내림 표기 누락 |
| **문항 설계 지침** | ✓ **오류 찾기(Error Analysis)** 유형 적극 활용<br>✓ "십의 자리에서 가져온 수는 실제로 얼마를 의미하는가?" → 답: 10<br>✓ 연속 받아내림 문항 (예: 300 - 157)<br>✓ 잘못된 계산 과정 제시 → 오류 단계 찾기 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 기본 계산 | 세 자리 수 덧셈·뺄셈 | 하 |
| 받아올림 과정 | 중간 과정 빈칸 채우기 | 중 |
| 받아내림 과정 | 단계별 계산 서술 | 중 |
| 오류 찾기 | 잘못된 계산에서 오류 찾기 | 중상 |
| 연속 받아내림 | 0을 포함한 뺄셈 | 상 |

### 2단원: 평면도형 - 정의의 엄밀성

> 직관적 도형 인식에서 수학적 정의로 전환되는 단원. 선분·반직선·직선, 각의 개념을 엄밀하게 도입.

#### 세부 개념 01: 선분·반직선·직선 (`concept-e3-plane-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **선분**: 두 점을 잇는 가장 짧은 선 (길이 유한, 양 끝점 있음)<br>- **반직선**: 한 점에서 시작해 한쪽으로 끝없이 뻗은 선 (시작점+방향)<br>- **직선**: 양쪽으로 끝없이 뻗은 선 (끝점 없음)<br>- 표기법: 선분 AB, 반직선 AB, 직선 AB |
| **주요 오개념** | - **반직선 동일성 오해** (가장 빈출): 반직선 AB = 반직선 BA라고 착각<br>  (선분은 양 끝점 같으면 같지만, 반직선은 시작점+방향 모두 같아야 함)<br>- **직선의 무한성 이해 부족**: 직선이 종이 밖으로도 계속 뻗어나간다는 것 이해 못함<br>- 선분·반직선·직선의 차이 혼동 |
| **문항 설계 지침** | ✓ 반직선 AB와 BA를 각각 다른 색으로 칠하기<br>✓ 주어진 그림에서 선분·반직선·직선 찾기<br>✓ 표기법 정확히 사용하기<br>✓ "선분 AB와 선분 BA는 같은가?" |

#### 세부 개념 02: 각과 직각 (`concept-e3-plane-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **각**: 한 점에서 그은 두 반직선으로 이루어진 도형 (꼭짓점, 변)<br>- **직각**: 종이를 반듯하게 두 번 접었을 때 생기는 각<br>- **직각 찾기**: 삼각자나 직각 모형 사용<br>- 다양한 방향의 각 인식 |
| **주요 오개념** | - **직각의 방향성 고착** (가장 빈출): 수평/수직으로 놓인 직각만 직각으로 인식<br>  (기울어진 직각삼각형에서 직각 못 찾음)<br>- **변 길이 = 각 크기 혼동**: 변이 길면 각도 크다고 착각<br>- 직각의 정의 모호: "대략 90도" 정도로만 이해 |
| **문항 설계 지침** | ✓ 다양한 각도로 회전된 도형에서 직각 모두 찾기<br>✓ 변의 길이는 다르지만 각의 크기는 같은 두 각 비교<br>✓ 직각 모형 활용: 종이 접기, 삼각자 대기<br>✓ "각 ㄱㄴㄷ의 꼭짓점은?" 용어 확인 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 선분·반직선·직선 구분 | 정의에 맞는 것 찾기 | 하 |
| 반직선 방향 | AB와 BA 구분 | 중 |
| 직각 찾기 | 회전된 도형에서 직각 찾기 | 중 |
| 변 길이와 각 크기 | 변이 달라도 각 같은 경우 | 중상 |
| 용어 확인 | 꼭짓점·변 용어 정확히 사용 | 하 |

### 3단원: 나눗셈 - 나눗셈의 두 가지 의미

> 나눗셈의 등분제와 포함제 상황을 구별하고, 곱셈과의 역연산 관계를 이해하는 단원.

#### 세부 개념 01: 등분제와 포함제 (`concept-e3-div1-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **등분제(Partitive Division)**: 전체를 똑같은 수의 묶음으로 나눌 때 한 묶음의 개수<br>  (예: 사탕 12개를 3명에게 똑같이 나누기 → 한 명당 4개)<br>- **포함제(Quotitive Division)**: 전체를 한 묶음씩 덜어낼 때 묶음의 개수<br>  (예: 사탕 12개를 3개씩 나누기 → 4묶음)<br>- 두 상황 모두 $12 \div 3 = 4$로 표현 |
| **주요 오개념** | - **몫의 의미 혼동** (가장 빈출): 몫 '4'가 '사람 수'인지 '사탕 개수'인지 단위 헷갈림<br>  (등분제/포함제 상황 구별 못함)<br>- **상황과 식의 연결 실패**: 구체물 그림을 보고 적절한 나눗셈식 세우기 못함<br>- "나누어지는 수가 항상 커야 한다" 오해 |
| **문항 설계 지침** | ✓ 구체물 그림 제시 → 상황에 맞는 나눗셈식 고르기<br>✓ 나눗셈식 주고 → 등분제/포함제 상황 구별<br>✓ "사과 4개를 12명에게 나누는 상황" 가능 여부 묻기<br>✓ 두 상황 모두 만들기: $12 \div 3$ 등분제·포함제 예 각각 |

#### 세부 개념 02: 곱셈과 나눗셈의 관계 (`concept-e3-div1-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **역연산 관계**: $12 \div 3 = 4$ ↔ $3 \times 4 = 12$<br>- 나눗셈 = 곱셈의 역연산<br>- **관계 짓기**: 하나의 그림이나 상황에서 곱셈식 2개, 나눗셈식 2개 만들기<br>- 곱셈구구로 나눗셈 결과 확인 |
| **주요 오개념** | - **교환법칙 적용 오류** (가장 빈출): $12 \div 4$와 $4 \div 12$를 같다고 착각<br>  (덧셈·곱셈과 달리 나눗셈은 교환법칙 성립 안 함)<br>- 곱셈식으로 나눗셈 검산하는 방법 모름<br>- 나눗셈을 곱셈으로 변환하지 못함 |
| **문항 설계 지침** | ✓ **관계 짓기 문항**: "그림을 보고 곱셈식 2개, 나눗셈식 2개 만드시오"<br>✓ 교환법칙 오류 확인: "$12 \div 3$과 $3 \div 12$는 같은가?"<br>✓ 곱셈구구로 나눗셈 결과 찾기<br>✓ 빈칸 채우기: $\square \div 3 = 4$ → $3 \times 4 = \square$ |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 등분제 상황 | 똑같이 나누는 상황 | 하 |
| 포함제 상황 | 몇 개씩 나누는 상황 | 하 |
| 상황 구별 | 등분제/포함제 구분 | 중 |
| 관계 짓기 | 곱셈식·나눗셈식 연결 | 중상 |
| 교환법칙 오류 | 나눗셈은 교환법칙 안 됨 | 중상 |

### 4단원: 곱셈 - 곱셈 알고리즘의 기초

> (몇십몇) × (몇) 계산을 통해 곱셈 알고리즘의 기초를 다지고, 분배법칙의 직관을 형성하는 단원.

#### 세부 개념 01: (몇십몇)×(몇) 기본 (`concept-e3-mul1-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - (몇십몇) × (몇) 계산 원리<br>- **분배법칙의 기초**: $23 \times 2 = 20 \times 2 + 3 \times 2 = 40 + 6 = 46$<br>- 곱셈 = 동수누가(Repeated Addition)의 효율적 표현<br>- 모눈종이(Area Model) 활용한 시각적 이해 |
| **주요 오개념** | - **자릿수 혼동** (가장 빈출): $23 \times 4$에서 $3 \times 4 = 12$, $2 \times 4 = 8$을 나열해 812로 표기<br>  (십의 자리 '2'가 실제로는 '20'임을 간과)<br>- 올림 없는 경우와 올림 있는 경우 혼동<br>- 동수누가와 곱셈의 관계 이해 부족 |
| **문항 설계 지침** | ✓ 모눈종이(Area Model) 활용: $20 \times 4$ 영역과 $3 \times 4$ 영역 구분<br>✓ 분배법칙 과정 단계별 서술<br>✓ 자릿값 강조 문항: "23의 2는 실제로 얼마를 나타내는가?"<br>✓ 동수누가로 곱셈 확인 |

#### 세부 개념 02: 분배법칙과 올림 (`concept-e3-mul1-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - 올림이 있는 (몇십몇) × (몇) 계산<br>- 분배법칙 적용 후 올림 처리<br>- 세로셈 알고리즘: 일의 자리부터 계산, 올림 표시<br>- 어림으로 결과 합리성 확인 |
| **주요 오개념** | - 올림 처리 실수: 올림한 수를 다음 계산에 더하는 것 누락<br>- 세로셈에서 자릿수 정렬 오류<br>- 결과 어림 능력 부족: 계산 실수 발견 못함 |
| **문항 설계 지침** | ✓ 올림 과정 빈칸 채우기<br>✓ 잘못된 세로셈에서 오류 찾기<br>✓ 어림하기: "$30 \times 4 = 120$이므로 $27 \times 4$의 결과는 120 근처"<br>✓ 곱셈 결과의 합리성 판단 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 기본 계산 | (몇십몇)×(몇) 계산 | 하 |
| 분배법칙 | 과정 단계별 서술 | 중 |

#### 세부 개념 01: 길이 단위와 변환 (`concept-e3-length-time-01`)

| 구분 | 내용 |

| **핵심 개념** | - **길이 단위**: 1mm, 1cm, 1m, 1km 도입 및 변환<br>  (1cm = 10mm, 1m = 100cm, 1km = 1000m)<br>- 자 사용법: 0 눈금에서 시작 또는 끝 눈금 - 시작 눈금<br>- 단위 환산: 큰 단위 → 작은 단위 (곱하기), 작은 단위 → 큰 단위 (나누기)<br>- 길이 어림 및 측정 |

### 6단원: 분수와 소수 - 새로운 수의 세계 ⭐ (최대 난관)

> 자연수에서 유리수로의 확장. 전 세계적으로 가장 많은 학생들이 어려워하는 개념.

#### 세부 개념 01: 분수와 단위분수 (`concept-e3-frac-dec-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **분수의 정의**: 전체를 '똑같이' 나눈 것 중의 일부분<br>- **단위분수**: 분자가 1인 분수 ($\frac{1}{2}, \frac{1}{3}, \frac{1}{4}$...)<br>- 분자: 색칠된 부분의 수, 분모: 전체를 똑같이 나눈 수<br>- 크기 비교: 분모가 같은 분수, 단위분수 |
| **주요 오개념** | - **'똑같이 나누기' 실패** (가장 빈출): 크기가 다른 4조각 중 1개를 색칠하고 $\frac{1}{4}$이라 함<br>  (이산량적 개수 세기 경험이 연속량 면적에 간섭)<br>- **단위분수의 역설**: $\frac{1}{3} > \frac{1}{2}$라고 착각<br>  (원인: 자연수 지식의 간섭 - 3이 2보다 크니까)<br>- 분수 표기법 혼동: 분자/분모 위치 |
| **문항 설계 지침** | ✓ 등분된 것과 안 된 것 섞어서 "$\frac{1}{4}$을 나타내는 그림 모두 고르기"<br>✓ 피자 2판: 2명 나눠 먹기 vs 3명 나눠 먹기 비교<br>  → "누가 더 많이 먹었는가?" (단위분수 역설 교정)<br>✓ 단위분수 크기 비교: 수직선 활용<br>✓ 분수 표기법 정확히 쓰기 |

#### 세부 개념 02: 소수와 분수-소수 관계 (`concept-e3-frac-dec-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **소수 도입**: $\frac{1}{10} = 0.1$ (일 분의 일)<br>- **소수점**: 일의 자리와 소수 첫째 자리를 구분<br>- 분수를 소수로: $\frac{3}{10} = 0.3$<br>- 소수를 분수로: $0.7 = \frac{7}{10}$<br>- **수직선 위 소수**: 0과 1 사이를 10칸으로 나누어 위치 표시 |
| **주요 오개념** | - **소수 0.1의 실체성 부재**: 0.1을 0보다 작은 수로 오해<br>- **소수점 무시**: 3.1을 31로 읽음<br>- 분수와 소수의 관계 이해 부족<br>- 0.10과 0.1의 관계 혼동 |
| **문항 설계 지침** | ✓ **수직선** 위에서 0과 1 사이를 10칸으로 나누고 0.1, 0.5 위치 찍기<br>✓ 분수-소수 변환 문제<br>✓ 소수의 크기 비교: 0.3과 0.7<br>✓ 실생활 맥락: 자의 mm 눈금과 소수 연결 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 분수 표현 | 그림에서 분수 찾기 | 하 |
| 똑같이 나누기 | 등분 여부 판단 | 중 |
| 단위분수 크기 | $\frac{1}{2}$과 $\frac{1}{3}$ 비교 | 중상 |
| 분수-소수 변환 | $\frac{3}{10} = 0.3$ | 중 |
| 수직선 위치 | 소수를 수직선에 표시 | 상 |

### 7단원: 곱셈 (2) - 알고리즘의 완성

> 세 자리 수와 두 자리 수의 곱셈으로 초등 연산 중 가장 복잡한 알고리즘을 완성하는 단원.

#### 세부 개념 01: (세 자리)×(한 자리) (`concept-e3-mul2-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - (세 자리 수) × (한 자리 수) 세로셈 알고리즘<br>- 일의 자리부터 순서대로 곱하기<br>- 각 자리에서 올림 처리<br>- 부분곱의 의미 이해 |
| **주요 오개념** | - 올림 처리 실수: 올림한 수를 다음 계산에 더하는 것 누락<br>- 0이 포함된 수의 곱셈 오류<br>- 자릿수 정렬 오류 |
| **문항 설계 지침** | ✓ 세로셈 과정 단계별 서술<br>✓ 0이 포함된 곱셈: 302 × 4<br>✓ 올림 과정 빈칸 채우기<br>✓ 어림으로 결과 합리성 확인 |

#### 세부 개념 02: (두 자리)×(두 자리) (`concept-e3-mul2-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - (두 자리 수) × (두 자리 수): 4번의 부분 곱(Partial Products) 합산<br>- **자릿수 맞추기**: 십의 자리로 곱한 부분곱은 한 칸 왼쪽에 정렬<br>- **핵심**: $34 \times 12 = 34 \times 10 + 34 \times 2 = 340 + 68$<br>- 초등 연산 중 가장 복잡한 알고리즘 |
| **주요 오개념** | - **자릿수 맞춤 오류 심화** (가장 빈출): $34 \times 12$에서<br>  $34 \times 10 = 340$을 써야 하는데 '0' 생략하고 34로 적음<br>  → $34 \times 2 + 34 \times 1$로 잘못 계산<br>- 부분곱의 의미 이해 부족: 왜 두 줄로 나누어 쓰는지 모름<br>- 올림 처리 중복 오류 |
| **문항 설계 지침** | ✓ 계산 과정에서 0을 쓰도록 '보조선'이나 '칸' 제공<br>✓ 부분곱의 의미 설명 문항<br>✓ 잘못된 세로셈에서 오류 찾기<br>✓ 어림하기: "30 × 10 = 300이므로 34 × 12는 300 근처" |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 세 자리×한 자리 | 기본 계산 | 하 |
| 올림 처리 | 올림 과정 확인 | 중 |
| 두 자리×두 자리 | 부분곱 합산 | 중상 |
| 자릿수 맞추기 | 0 쓰기 강제 | 상 |
| 오류 찾기 | 잘못된 계산 수정 | 상 |

### 8단원: 나눗셈 (2) - 나머지의 발견

> 나누어떨어지지 않는 경우를 다루며, 나머지의 맥락적 해석 능력을 기르는 단원.

#### 세부 개념 01: 나머지와 검산 (`concept-e3-div2-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **나머지가 있는 나눗셈**: 나누어떨어지지 않는 경우<br>- **검산**: $A \div B = Q ... R$ → $A = B \times Q + R$<br>- **제약 조건**: 나머지 < 나누는 수 ($R < B$)<br>- 나머지의 의미: 더 나눌 수 없는 남은 수 |
| **주요 오개념** | - **나머지 크기 조건 위반** (가장 빈출): $14 \div 3 = 3 ... 5$라고 씀<br>  (덜어낼 수 있을 때까지 덜어내야 함을 모름)<br>- 검산 공식 적용 미숙<br>- 나머지를 항상 버리거나 무시 |
| **문항 설계 지침** | ✓ 잘못된 계산 제시 → "나머지의 성질을 이용해 틀린 이유 설명하기"<br>✓ 검산으로 나눗셈 결과 확인하기<br>✓ 나머지 조건 확인: "나머지는 나누는 수보다 작아야 한다"<br>✓ 몫과 나머지 동시에 구하기 |

#### 세부 개념 02: 나머지의 맥락적 해석 (`concept-e3-div2-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - 실생활 문장제에서 나머지 처리 판단<br>- **올림 상황**: 남는 사람도 텐트 필요<br>  (예: 32명을 5명씩 텐트 배정 → 7텐트, 남은 2명 위해 1개 더 필요 → 8텐트)<br>- **버림 상황**: 완성품만 인정<br>  (예: 14개로 3개씩 묶음 만들기 → 4묶음, 남은 2개는 버림)<br>- **나머지가 답인 상황**: 남는 개수 구하기 |
| **주요 오개념** | - **맥락적 해석 실패** (가장 빈출): 나머지를 항상 버리거나 항상 올림<br>- 몫과 나머지 중 무엇이 답인지 혼동<br>- 실생활 상황과 수학적 계산 연결 못함 |
| **문항 설계 지침** | ✓ **올림 상황** vs **버림 상황** 구별 문항 세트 출제<br>✓ 나머지가 답인 상황: "몇 개가 남는가?"<br>✓ 같은 계산 다른 상황: $14 \div 3 = 4...2$를 다른 맥락에 적용<br>✓ 상황 판단 서술형: "왜 올림/버림했는지 설명하시오" |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 나머지 기본 | 나머지 있는 나눗셈 | 하 |
| 검산 | 나눗셈 결과 확인 | 중 |
| 나머지 조건 | 나머지 < 나누는 수 | 중 |
| 올림 상황 | 나머지 처리: 올림 | 중상 |
| 버림 상황 | 나머지 처리: 버림 | 중상 |

### 9단원: 원 - 원의 기본 성질

> 원의 구성 요소를 학습하고, 반지름과 지름의 관계를 이해하는 단원.

#### 세부 개념 01: 중심·반지름·지름 정의 (`concept-e3-circle-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **원의 중심**: 원 그릴 때 고정된 점<br>- **반지름**: 중심에서 원 위의 한 점까지 선분 (한 원에서 모두 같음)<br>- **지름**: 원 위의 두 점을 잇는 선분 중 **중심을 지나는** 선분<br>- 컴퍼스 사용법: 중심 고정, 반지름 조절 |
| **주요 오개념** | - **반지름 방향 가변성 오해**: 위쪽/옆쪽 반지름 길이가 다르다고 착각<br>  (시각적으로 길어 보이는 것을 더 길다고 판단)<br>- **지름 정의 오류**: 원 내부를 지나는 아무 선분(현)을 지름이라 부름<br>  → 지름은 **반드시 중심을 지나야** 함<br>- 컴퍼스 사용 시 중심점 고정 못함 |
| **문항 설계 지침** | ✓ 찌그러진 원(타원) vs 바른 원: "컴퍼스로 그린 원은? 이유는?"<br>✓ 원 안에 여러 선분 그려놓고 지름인 것/아닌 것 분류<br>  (기준: **'원의 중심을 지나는가?'**)<br>✓ 반지름 길이 측정: 다양한 방향 모두 같음 확인<br>✓ 컴퍼스로 원 그리기 활동 |

#### 세부 개념 02: 반지름-지름 관계와 활용 (`concept-e3-circle-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **지름 = 반지름 × 2**: 지름은 반지름의 2배<br>- **반지름 = 지름 ÷ 2**: 반지름은 지름의 절반<br>- 원의 크기 비교: 반지름(또는 지름)이 클수록 큰 원<br>- 실생활 활용: 원 모양 물건 측정 |
| **주요 오개념** | - 지름과 반지름의 관계 암기만 하고 의미 이해 못함<br>- 원의 크기를 반지름이 아닌 다른 요소로 판단<br>- 지름을 잴 때 중심 안 지나는 선분 측정 |
| **문항 설계 지침** | ✓ 반지름 주고 지름 구하기, 지름 주고 반지름 구하기<br>✓ 원의 크기 비교: 여러 원 중 가장 큰/작은 것<br>✓ 실생활 원 모양: 시계, 접시 등의 지름 구하기<br>✓ 지름 측정 방법: 자로 중심 지나게 재기 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 중심·반지름·지름 | 정의 확인 | 하 |

### 10단원: 분수 (2) - 분수의 다양한 형태

> 진분수·가분수·대분수를 도입하고, 분모가 같은 분수의 연산을 다루는 단원.

#### 세부 개념 01: 진분수·가분수·대분수 (`concept-e3-frac2-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **진분수**: 분자 < 분모 (1보다 작은 수)<br>- **가분수**: 분자 ≥ 분모 (1보다 크거나 같은 수)<br>- **대분수**: 자연수 + 진분수<br>- 대분수 ↔ 가분수 변환<br>  ($1\frac{2}{3} = \frac{5}{3}$, $\frac{7}{4} = 1\frac{3}{4}$) |
| **주요 오개념** | - 가분수를 잘못된 분수라고 착각<br>- 대분수 ↔ 가분수 변환 오류<br>- 분수의 크기와 형태 혼동 |
| **문항 설계 지침** | ✓ 진분수·가분수·대분수 분류<br>✓ 대분수 → 가분수 변환: 단계별 과정<br>✓ 가분수 → 대분수 변환<br>✓ 수직선에 가분수·대분수 위치 표시 |

#### 세부 개념 02: 동분모 비교와 연산 (`concept-e3-frac2-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **분모가 같은 분수의 크기 비교**: 분자가 큰 것이 큰 수<br>- **분모가 같은 분수의 덧셈과 뺄셈**: 분모 그대로, 분자끼리 연산<br>  $\frac{2}{7} + \frac{3}{7} = \frac{5}{7}$<br>- 파이 차트/막대 모델로 시각적 증명 |
| **주요 오개념** | - **분모 덧셈 오류** (가장 빈출): $\frac{2}{7} + \frac{3}{7} = \frac{5}{14}$<br>  (원인: 자연수 덧셈 도식의 간섭 - 분모와 분자 각각 더함)<br>- 분모가 같다는 것의 의미(같은 단위) 이해 못함<br>- 뺄셈 결과가 0이 되는 경우 처리 혼란 |
| **문항 설계 지침** | ✓ 파이 차트 그림으로 $\frac{2}{7}$ + $\frac{3}{7}$ 시각적 증명<br>  → 조각의 크기(분모)는 불변, 개수(분자)만 증가<br>✓ 오류 수정형: 분모 덧셈 오류 찾기<br>✓ 동분모 분수 크기 비교<br>✓ 단위의 통일성 설명: "같은 크기 조각끼리 더하기" |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 분수 분류 | 진분수·가분수·대분수 | 하 |
| 변환 | 대분수↔가분수 | 중 |
| 크기 비교 | 동분모 분수 비교 | 중 |
| 동분모 덧셈 | 분자끼리 더하기 | 중 |
| 오류 수정 | 분모 덧셈 오류 찾기 | 중상 |

### 11단원: 들이와 무게 - 측정의 확장

> 들이와 무게의 단위를 학습하고, 부피와 무게의 차이를 이해하는 단원.

#### 세부 개념 01: 단위와 변환 (`concept-e3-vol-wt-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **들이 단위**: L(리터), mL(밀리리터) (1L = 1000mL)<br>- **무게 단위**: kg(킬로그램), g(그램) (1kg = 1000g)<br>- 단위 환산: 큰 단위 → 작은 단위 (곱하기), 작은 단위 → 큰 단위 (나누기)<br>- 측정 도구: 눈금 실린더, 저울 |
| **주요 오개념** | - 단위 환산 오류: 곱하기/나누기 방향 혼동<br>- L와 mL, kg과 g 혼동<br>- 1000배 관계를 100배로 착각 |
| **문항 설계 지침** | ✓ 단위 환산 문제: 2L 300mL = ?mL<br>✓ 측정 도구 눈금 읽기<br>✓ 실생활 물건의 들이·무게 어림<br>✓ 단위 적절히 선택하기 |

#### 세부 개념 02: 부피≠무게와 덧뺄셈 (`concept-e3-vol-wt-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **들이와 무게의 독립성**: 부피가 크다고 무게가 큰 것 아님<br>- 같은 단위끼리 덧셈과 뺄셈<br>- 받아올림·받아내림: 1000이 기준<br>- 실생활 문제해결 |
| **주요 오개념** | - **부피 = 무게 혼동** (가장 빈출): 부피가 크면 무조건 더 무겁다고 착각<br>  (예: 솜뭉치 vs 쇠구슬)<br>- 단위가 다른 것끼리 계산 (1L + 500g 같은 오류)<br>- 받아올림·받아내림에서 10이 아닌 1000 기준 적용 못함 |
| **문항 설계 지침** | ✓ 시각적 부피와 실제 무게가 상충하는 상황 제시<br>  (예: "스티로폼 상자 vs 작은 아령, 어느 것이 더 무거운가?")<br>✓ 같은 단위끼리 덧뺄셈 문항<br>✓ 단위가 다른 것끼리 계산 오류 찾기<br>✓ 받아올림·받아내림 (1000 기준) |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 단위 환산 | 들이·무게 단위 변환 | 하 |
| 눈금 읽기 | 측정 도구 읽기 | 중 |
| 부피≠무게 | 독립성 이해 | 중상 |
| 같은 단위 계산 | 덧뺄셈 | 중 |
| 받아올림·받아내림 | 1000 기준 | 중상 |

## 문제 개발 전략

### 1. 오류 찾기(Error Analysis) 문항 활용

> 가상의 학생이 푼 **틀린 풀이 과정** 제시
> - "왜 틀렸는지 설명하시오"
> - "어느 단계에서 처음 실수가 발생했는지 찾으시오"

**효과**: 메타인지 능력 평가에 탁월

| 나쁜 예 | 좋은 예 |
|---------|---------|
| "$423-157$을 계산하시오." | "민수는 $423-157$을 계산하면서 일의 자리에서 $7-3=4$로 계산했습니다. 틀린 이유를 설명하시오." |
| "$\frac{2}{7} + \frac{3}{7}$을 계산하시오." | "$\frac{2}{7} + \frac{3}{7} = \frac{5}{14}$이 틀린 이유를 그림으로 설명하시오." |

| 분모끼리 더하기 | 파이 조각 그림 | 조각 크기 불변, 개수만 증가 |

---

### 5. 2022 개정 교육과정 3대 원칙

| 원칙 | 내용 |
|------|------|
| **과정 중심 평가** | 결과값만 묻지 않고, 계산의 원리나 도형의 정의를 서술하게 함 |
| **디지털 소양 반영** | 태블릿 자/각도기 앱 사용 상황을 문제 맥락으로 제시 |
| **연결성 강조** | 분수와 나눗셈, 곱셈과 면적 등 단원 간 융합형 문항 설계 |

## 핵심 원칙

> **3학년 수학의 본질**
>
> 직관적·구체적 활동에서 **형식적 알고리즘과 엄밀한 정의**로 전환되는 결정적 시기

### 학기별 핵심 키워드

| 학기 | 핵심 |
|------|------|
| **1학기** | 자연수 연산 완성, 도형의 **정의** 도입, **분수·소수** 첫 만남 |
| **2학기** | 곱셈 알고리즘 완성, **나머지** 개념, **원**의 성질, 분수 확장 |

### 3학년 분수 지도의 핵심 3가지

1. **'똑같이 나누기'의 엄격한 적용** (등분 vs 비등분 구별)
2. **단위분수의 역설 극복** (분모가 클수록 조각은 작다)
3. **수직선을 통한 소수의 '수'로서의 정체성 확립**

### 2022 개정 핵심 변화 3가지

1. **5대 핵심 역량 재구조화** → 문제해결·추론·의사소통·연결·정보처리
2. **개념적 이해 강화** → 단순 계산 숙달 지양, 원리 이해 강조
3. **과정 중심 평가** → 결과보다 사고 과정 추적, 서술형 강화
