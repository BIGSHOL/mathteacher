# 초등학교 4학년 수학 개념 문제 개발 가이드
> 2022 개정 교육과정 기반 | 심층 분석 버전 (12개념 -> 24개념 세분화)

---

## 초4 수학의 특징

- **위상**: 구체적 조작기 -> 형식적 조작기로 넘어가는 **과도기(Critical Transition Period)**
- **인지 발달**: 추상화 수준이 급격히 높아지는 시기
- **학문적 전환**:
  - 단순 연산 숙달 -> **수학적 문해력과 개념적 이해** 강조
  - 자연수 범위 확장(만, 억, 조) -> **큰 수 감각** 형성
  - 직관적 도형 인식 -> **도형의 분류와 성질** 탐구 시작
  - 자료 정리 -> **그래프 해석 및 작성** 능력 도입
- **2022 개정 핵심 변화**:
  - **5대 핵심 역량**: 문제해결, 추론, 의사소통, 연결, 정보처리로 재구조화
  - **수학적 모델링**: 실생활 문제를 수학적으로 변환/해결/해석
  - **과정 중심 평가**: 결과 중심 -> 사고 과정 추적, 서술형 강화
  - **디지털 소양**: 공학 도구 활용 강조

> 개념 문제는 단순 연산 숙달이 아닌 **'수학적 문해력(Mathematical Literacy)'** 함양에 초점

## 1학기 단원별 정리

> 1학기는 큰 수로 수 감각을 확장하고, 각도로 측정의 기초를 다지며, 곱셈과 나눗셈으로 연산 능력을 심화하고, 평면도형의 이동으로 공간 감각을, 막대그래프로 자료 해석력을, 규칙 찾기로 대수적 사고의 기초를 형성하는 과정이다.

### 1단원: 큰 수 - 수 체계의 확장

> 만, 억, 조 단위를 도입하여 십진기수법의 구조를 체계적으로 이해하고, 큰 수의 양감과 크기 비교 능력을 기르는 단원. 중학교 수 체계 확장의 기초.

#### 세부 개념 01: 십진기수법과 자릿값 (`concept-e4-big-num-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **10배의 원리**: 자릿수가 왼쪽으로 한 칸 -> 10배<br>- **4자리씩 끊어 읽기**: 만, 억, 조 단위 구분<br>- **자릿값**: 같은 숫자라도 위치에 따라 나타내는 값이 다름<br>  (예: 5555에서 각 5의 자릿값은 5000, 500, 50, 5)<br>- 뛰어 세기: 10만씩, 100만씩 뛰어 세기 |
| **주요 오개념** | - **0이 포함된 수 표기 오류** (가장 빈출): "삼억 오천만" -> 35000000으로 표기 (3억=300000000에서 0 누락)<br>- **자릿값과 자릿수 혼동**: 5가 나타내는 값(자릿값) != 숫자 5 자체<br>- 큰 수의 양감 부족: 만, 억, 조의 크기 차이 체감 못함<br>- 4자리 끊어 읽기 규칙을 3자리(영어권)로 혼동 |
| **문항 설계 지침** | - 연속된 0이 있는 수의 받아쓰기 문항<br>- 특정 단위가 생략된 수 읽기/쓰기<br>- 자릿값 문제: "7이 나타내는 값이 70000인 수를 고르시오"<br>- 뛰어 세기 패턴: 빈칸 채우기 |

#### 세부 개념 02: 수의 크기 비교 (`concept-e4-big-num-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **크기 비교 전략**: 자릿수 비교 -> 같으면 최상위 자릿수부터 비교<br>- **양감 형성**: 국가 예산, 인구, 천체 거리 등 실생활 데이터 연결<br>- 수직선 위에 큰 수 위치 추정<br>- 크기순 배열: 여러 큰 수를 작은 수부터 또는 큰 수부터 |
| **주요 오개념** | - **자릿수가 같을 때 앞자리 비교 누락**: 무작정 각 자리를 비교하지 않고 전체적으로 판단 시도<br>- 0이 많은 수에서 자릿수 세기 오류<br>- 단위가 다른 수 비교 시 단위 통일 못함 (예: 3억 vs 28000만) |
| **문항 설계 지침** | - 자릿수가 같은 두 수의 크기 비교 (최상위부터 순차 비교)<br>- 단위 혼합 비교: "3억과 28000만 중 어느 것이 큰가?"<br>- 실생활 데이터: 국가 인구, 면적 등 비교<br>- 수직선 위 큰 수 위치 추정 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 수 읽기/쓰기 | 0이 포함된 큰 수 받아쓰기 | 하 |
| 자릿값 | 특정 숫자가 나타내는 값 | 중 |
| 뛰어 세기 | 큰 단위 뛰어 세기 패턴 | 중 |
| 크기 비교 | 단위 혼합된 큰 수 비교 | 중상 |
| 양감 형성 | 실생활 맥락의 큰 수 해석 | 상 |

### 2단원: 각도 - 측정의 기초

> 각의 크기를 정량적으로 측정하고 분류하는 능력을 기르는 단원. 도형의 성질 탐구와 중학교 기하학의 기초.

#### 세부 개념 01: 각의 크기와 분류 (`concept-e4-angle-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **각의 크기**: 두 변이 벌어진 정도 (변의 길이와 무관!)<br>- **각도기 사용**: 꼭짓점 맞추기, 한 변을 0에 정렬, 안쪽/바깥쪽 눈금 구분<br>- **각의 분류**:<br>  - 예각: 0도 초과 ~ 90도 미만<br>  - 직각: 정확히 90도<br>  - 둔각: 90도 초과 ~ 180도 미만<br>  - 평각: 정확히 180도 |
| **주요 오개념** | - **변이 길면 각도 크다고 착각** (가장 빈출): 각의 크기는 벌어진 정도이며 변의 길이와 무관<br>- **각도기 안쪽/바깥쪽 눈금 혼동**: 둔각(120도)을 60도로 읽음<br>- 직각의 정확한 정의 혼동: "대략 90도"와 "정확히 90도" 구분 미흡<br>- 각도 어림 능력 부족 |
| **문항 설계 지침** | - 변의 길이는 다르나 각이 같은 도형 비교<br>- 각도기 읽기: 안쪽/바깥쪽 눈금 구분 문항<br>- 각 분류: 주어진 각도를 예각/직각/둔각으로 분류<br>- 각도 어림 후 확인: 먼저 어림하고 각도기로 측정 |

#### 세부 개념 02: 내각의 합 (`concept-e4-angle-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **삼각형 내각의 합**: 항상 180도 (삼각형 모양에 무관)<br>- **사각형 내각의 합**: 항상 360도 (대각선으로 삼각형 2개)<br>- **활동**: 삼각형 세 각을 찢어 한 점에 모으면 일직선(180도) 형성<br>- 각도 구하기: 두 각을 알면 나머지 한 각 계산 |
| **주요 오개념** | - **삼각형 모양이 달라지면 내각의 합도 변한다고 오해**: 어떤 삼각형이든 180도<br>- 내각의 합 공식을 외우되 활동 경험 없이 납득하지 못함<br>- 사각형 내각의 합을 삼각형으로부터 유도하지 못함<br>- 나머지 각 구하기에서 뺄셈 오류 |
| **문항 설계 지침** | - 다양한 삼각형(예각/직각/둔각)의 내각의 합 확인 문항<br>- 두 각 제시 후 나머지 각 구하기<br>- 사각형을 삼각형으로 나누어 내각의 합 유도<br>- 활동형: 삼각형 세 각 찢어 모으기 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 각 분류 | 주어진 각을 예각/직각/둔각으로 분류 | 하 |
| 각도기 읽기 | 안쪽/바깥쪽 눈금 구분 | 중 |
| 변 길이와 각 | 변 길이 다르나 각 같은 도형 비교 | 중 |
| 나머지 각 구하기 | 내각의 합으로 미지각 계산 | 중상 |
| 내각의 합 유도 | 사각형 내각의 합을 삼각형으로 설명 | 상 |

### 3단원: 곱셈과 나눗셈 - 연산의 심화

> 세 자리 수와 두 자리 수의 곱셈/나눗셈으로 연산 알고리즘을 완성하는 단원. 나머지의 맥락적 해석까지 포함하여 수학적 문제해결력을 기르는 핵심 단원.

#### 세부 개념 01: 곱셈 알고리즘 (`concept-e4-mul-div-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **(세 자리 수) x (두 자리 수)** 세로셈 알고리즘<br>- 부분곱의 의미: 일의 자리로 곱한 값 + 십의 자리로 곱한 값<br>- **자릿수 맞추기**: 십의 자리로 곱한 부분곱은 한 칸 왼쪽에 정렬<br>- 어림: 곱셈 결과의 범위를 미리 예측 |
| **주요 오개념** | - **세로셈에서 자릿수 맞추기 오류** (가장 빈출): 십의 자리 부분곱을 일의 자리에 맞춰 씀<br>- 올림(받아올림) 처리 실수: 올림한 수를 다음 계산에 더하는 것 누락<br>- 부분곱의 의미 이해 부족: 왜 두 줄로 나누어 쓰는지 모름<br>- 결과 어림 능력 부족 |
| **문항 설계 지침** | - 세로셈 과정에서 부분곱의 의미 설명 문항<br>- 잘못된 세로셈에서 오류 찾기<br>- 어림하기: "300 x 20 = 6000이므로 327 x 18의 결과는 6000 근처"<br>- 곱셈 결과의 합리성 판단 |

#### 세부 개념 02: 나눗셈과 검산 (`concept-e4-mul-div-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **(세 자리 수) / (두 자리 수)**: 가늠 -> 곱하기 -> 빼기 -> 확인<br>- **몫 가늠하기**: 나누는 수를 반올림하여 어림 몫 구하기<br>- **검산**: 나누는 수 x 몫 + 나머지 = 나누어지는 수<br>- **나머지의 맥락적 해석**: 실생활에서 올림/버림 판단<br>  (예: 32명이 한 줄에 5명씩 -> 7줄이면 3명 남음 -> 8줄 필요) |
| **주요 오개념** | - **몫 가늠하기 어려움** (가장 빈출): 적절한 몫을 한 번에 찾지 못함<br>- 나머지가 나누는 수보다 큰 채로 답을 내림<br>- 검산 공식 적용 미숙<br>- **나머지의 맥락적 해석 실패**: 나머지를 항상 버리거나 항상 올림 |
| **문항 설계 지침** | - 몫의 범위 어림하기 문항<br>- 잘못된 계산 과정에서 오류 찾기<br>- 검산으로 결과 확인하기<br>- 실생활 문장제에서 나머지 처리 판단 (올림 vs 버림) |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 세로셈 계산 | 곱셈/나눗셈 세로셈 수행 | 하 |
| 부분곱 의미 | 부분곱이 무엇을 나타내는지 설명 | 중 |
| 오류 찾기 | 잘못된 계산에서 오류 단계 찾기 | 중상 |
| 검산 | 나눗셈 결과를 곱셈으로 확인 | 중 |
| 나머지 해석 | 실생활 상황에서 올림/버림 판단 | 상 |

### 4단원: 평면도형의 이동 - 공간 변환의 기초

> 도형의 밀기, 뒤집기, 돌리기를 통해 공간 감각을 기르고, 중학교 도형의 합동/대칭 학습의 기초를 형성하는 단원.

#### 세부 개념 01: 밀기 (`concept-e4-transform-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **밀기(Translation)**: 위치만 변화, 모양/크기/방향 모두 불변<br>- 방향과 거리: 밀기의 방향(상/하/좌/우/대각선)과 거리 명시<br>- **보존 성질**: 대응점 사이의 거리가 모두 같고, 대응변이 평행<br>- 모눈종이 위에서 점의 좌표 변화 추적 |
| **주요 오개념** | - 밀기와 돌리기를 혼동: 밀기 후에도 방향이 바뀐다고 착각<br>- 대각선 방향 밀기를 가로+세로 분해하지 못함<br>- "위치만 바뀐다"는 것의 의미를 완전히 이해하지 못함 |
| **문항 설계 지침** | - 모눈종이 위에서 도형 밀기 작도<br>- 밀기 전후 대응점의 좌표 변화 확인<br>- 밀기 후 모양/크기/방향 보존 확인 문항<br>- 대각선 방향 밀기: 가로/세로 분해 활동 |

#### 세부 개념 02: 뒤집기와 돌리기 (`concept-e4-transform-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **뒤집기(Reflection)**: 대칭축 기준 좌우/상하 반전<br>- **돌리기(Rotation)**: 중심점 기준 90도, 180도, 270도, 360도 회전<br>- 시계 방향 / 반시계 방향 구분<br>- **합성 변환**: 뒤집기 + 돌리기 복합 적용 |
| **주요 오개념** | - **오른쪽 뒤집기와 시계방향 180도 돌리기 혼동** (가장 빈출)<br>- **시계/반시계 방향 혼동**: 시계 방향 90도와 반시계 방향 270도가 같은 결과임을 모름<br>- 90도 돌리기와 270도 돌리기 결과 착각<br>- 비대칭 도형에서 변환 결과 예측 어려움 |
| **문항 설계 지침** | - **비대칭 도형(ㄱ, ㄴ, P, F 등) 활용 필수**: 대칭 도형은 변환 차이가 안 보임<br>- 뒤집기 + 돌리기 복합 적용 문제<br>- 기준선(변) 추적하는 분석적 접근 유도<br>- 결과 도형 보고 어떤 변환이 적용됐는지 역추론 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 밀기 작도 | 모눈종이 위 도형 밀기 | 하 |
| 뒤집기 기본 | 대칭축 기준 반전 | 중 |
| 돌리기 기본 | 중심점 기준 90도/180도/270도 회전 | 중 |
| 변환 구분 | 결과 보고 어떤 변환인지 판단 | 중상 |
| 복합 변환 | 뒤집기 + 돌리기 합성 | 상 |

> 범주형 자료를 막대그래프로 시각화하고 해석하는 능력을 기르는 단원. 통계적 소양의 기초.

| 구분 | 내용 |

| 해석 서술형 | 그래프 분석 후 결론 서술 | 중 |
| 눈금 설정 | 적절한 눈금 크기 결정 | 중상 |
| 그래프 작성 | 표를 보고 막대그래프 그리기 | 상 |

### 6단원: 규칙 찾기 - 대수적 사고의 싹

> 수와 도형의 배열에서 규칙을 발견하고 수학적 언어로 표현하는 단원. 중학교 함수 개념과 문자식의 기초.

#### 세부 개념 01: 수 배열 규칙 (`concept-e4-pattern-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **수 배열에서 규칙 찾기**: 덧셈, 뺄셈, 곱셈, 나눗셈 규칙<br>- **수 배열표 탐구**: 가로/세로/대각선 방향의 규칙 발견<br>- **규칙의 일반화**: 규칙을 말이나 식으로 표현<br>- 다음 수 예측: 발견한 규칙으로 빈칸이나 다음 항 구하기 |

| **문항 설계 지침** | - 바둑돌/성냥개비 패턴 -> "n번째 도형의 개수" 문항<br>- 계산식 패턴 발견 후 다음 식 예측<br>- 규칙의 변화를 설명하는 서술형<br>- 먼 단계 예측: 규칙을 이용한 효율적 풀이 유도 |

| 복합 규칙 | 증가하는 차이/곱셈 규칙 | 중 |
| 도형 수치화 | 도형 패턴을 수로 변환 | 중상 |

### 7단원: 분수의 덧셈과 뺄셈 - 동분모 연산의 기초

> 분모가 같은 분수의 연산을 다루며, 5학년 이분모 분수 연산의 필수 기초를 형성하는 단원. 분수 연산의 가장 보편적 오개념이 시작되는 시기.

#### 세부 개념 01: 동분모 진분수 연산 (`concept-e4-frac-op-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **동분모 분수 덧셈/뺄셈**: 분모는 그대로, 분자끼리 연산<br>  $\frac{2}{5} + \frac{1}{5} = \frac{3}{5}$<br>- **핵심 원리**: 분모 = 단위(조각의 크기), 분자 = 개수 -> 같은 단위끼리 연산<br>- **시각 모델**: 파이/막대 그림으로 연산 과정 확인<br>- 결과가 1이 넘으면 대분수 또는 가분수로 표현 |
| **주요 오개념** | - **분모끼리 더하기** (가장 빈출, 전 세계 공통): $\frac{1}{3} + \frac{1}{3} = \frac{2}{6}$<br>  -> 분수를 양(Quantity)이 아닌 두 자연수의 쌍으로 인식<br>- 분모가 같다는 것의 의미(같은 단위)를 이해 못함<br>- 뺄셈 결과가 0이 되는 경우 처리 혼란<br>- 결과를 기약분수로 만들지 않음 |
| **문항 설계 지침** | - 파이/막대 그림 모델과 식 연결 문항<br>- 오류 수정형: $\frac{1}{3} + \frac{1}{3} = \frac{2}{6}$이 틀린 이유 설명<br>- "분모가 같다"는 것의 의미 서술형<br>- 결과가 가분수가 되는 경우 대분수로 변환 |

#### 세부 개념 02: 대분수와 받아내림 (`concept-e4-frac-op-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **대분수 덧셈**: 자연수끼리 + 분수끼리 (또는 가분수로 변환)<br>- **대분수 뺄셈 받아내림**: 분수 부분에서 뺄 수 없으면 자연수에서 1을 빌려옴<br>  $3\frac{1}{5} - 1\frac{3}{5}$ -> $2\frac{6}{5} - 1\frac{3}{5} = 1\frac{3}{5}$<br>- **핵심**: 빌려온 1 = $\frac{\text{분모}}{\text{분모}}$ (분모가 5이면 $\frac{5}{5}$)<br>- 자연수 - 분수 계산: $3 - \frac{2}{5} = 2\frac{3}{5}$ |
| **주요 오개념** | - **받아내림 시 분자에 10 더함** (가장 빈출): 십진법 습관이 전이됨<br>  (정답: 분모만큼 더해야 함. 분모가 5이면 $\frac{5}{5}$를 빌려옴)<br>- 자연수 - 분수 계산 오류: $3 - \frac{2}{5}$ 처리 못함<br>- 대분수를 가분수로 변환하지 않고 곧바로 계산하려다 오류<br>- 답을 대분수로 정리하지 않음 |
| **문항 설계 지침** | - 받아내림 과정 단계별 서술 문항<br>- 자연수를 가분수로 바꾸는 재그룹화 과정 시각화<br>- 틀린 계산 과정 제시 -> 오류 찾기(Error Correction)<br>- 자연수 - 분수 형태 문제 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 동분모 기본 | 분모 같은 진분수 덧뺄셈 | 하 |
| 그림-식 연결 | 파이/막대 모델과 식 연결 | 중 |
| 대분수 덧셈 | 자연수끼리+분수끼리 | 중 |
| 받아내림 | 자연수에서 빌려와 계산 | 중상 |
| 오류 수정 | 잘못된 받아내림에서 오류 찾기 | 상 |

### 8단원: 삼각형 - 도형 분류의 시작

> 삼각형을 변과 각에 따라 분류하고, 도형 간의 포함 관계를 이해하는 단원. 논리적 분류 능력과 도형의 성질 탐구의 기초.

#### 세부 개념 01: 변에 의한 분류 (`concept-e4-triangle-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **이등변삼각형**: 두 변의 길이가 같음, 두 밑각의 크기가 같음<br>- **정삼각형**: 세 변의 길이가 모두 같음, 세 각이 모두 60도<br>- **포함 관계**: 정삼각형 $\subset$ 이등변삼각형<br>  (정삼각형은 이등변삼각형의 특수한 경우)<br>- 다양한 방향의 도형 인식: 회전된 이등변삼각형도 이등변삼각형 |
| **주요 오개념** | - **"정삼각형은 이등변삼각형인가?" -> "아니오" 오답** (포함 관계 이해 부족)<br>- 회전된 이등변삼각형 인식 못함: 밑변이 아래에 있는 형태만 이등변삼각형으로 인식<br>- 이등변삼각형의 두 밑각이 같다는 성질 모름<br>- 눈대중으로 변의 길이 판단하여 오류 |
| **문항 설계 지침** | - 포함 관계 확인: "정삼각형은 이등변삼각형인가?"<br>- 다양한 방향의 도형 제시: 회전된 이등변삼각형 식별<br>- 변의 길이 조건에 맞는 삼각형 그리기<br>- 이등변삼각형의 성질(밑각) 활용 문항 |

#### 세부 개념 02: 각에 의한 분류 (`concept-e4-triangle-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **예각삼각형**: 세 각이 모두 예각 (모두 90도 미만)<br>- **직각삼각형**: 한 각이 직각 (정확히 90도)<br>- **둔각삼각형**: 한 각이 둔각 (90도 초과)<br>- **핵심 판별**: 가장 큰 각 하나로 분류가 결정됨<br>- 변에 의한 분류와 각에 의한 분류는 **독립적** (교차 분류 가능) |
| **주요 오개념** | - **예각이 하나만 있어도 예각삼각형이라 착각** (가장 빈출):<br>  둔각삼각형에도 예각 2개 존재 -> 예각이 있다고 예각삼각형 아님!<br>  예각삼각형 = 세 각 **모두** 예각<br>- 둔각삼각형에서 예각을 발견하고 "예각삼각형"이라 답함<br>- 직각삼각형에서 직각이 아닌 두 각의 합이 90도임을 모름 |
| **문항 설계 지침** | - 둔각삼각형에서 예각 찾고 오답 유도: "이 삼각형은 예각삼각형인가?"<br>- 다양한 방향의 도형 제시<br>- 세 각의 크기 주고 삼각형 종류 분류<br>- 변에 의한 분류와 각에 의한 분류 교차: "이등변 둔각삼각형" 가능 여부 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 변 분류 기본 | 이등변/정삼각형 식별 | 하 |
| 각 분류 기본 | 예각/직각/둔각삼각형 식별 | 중 |
| 포함 관계 | 정삼각형이 이등변삼각형인지 | 중 |
| 둔각삼각형 함정 | 예각이 있지만 예각삼각형 아닌 경우 | 중상 |
| 교차 분류 | 변과 각 분류 동시 적용 | 상 |

### 9단원: 소수의 덧셈과 뺄셈 - 소수 연산 체계의 기초

> 소수의 구조를 이해하고 크기를 비교하며, 소수의 덧뺄셈 알고리즘을 습득하는 단원. 5학년 소수 곱셈의 기초.

#### 세부 개념 01: 소수 구조와 크기 비교 (`concept-e4-dec-op-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **소수의 자릿값**: 0.1, 0.01, 0.001의 의미<br>  (1 = 0.1 x 10 = 0.01 x 100 = 0.001 x 1000)<br>- **소수 크기 비교**: 소수점 앞부터 차례로, 같은 자리끼리 비교<br>- **동치 소수**: 3.5 = 3.50 = 3.500 (뒤에 0을 붙여도 크기 불변)<br>- 소수를 분수로 변환: 0.25 = $\frac{25}{100}$ = $\frac{1}{4}$ |
| **주요 오개념** | - **소수가 길면 크다고 착각** (가장 빈출): 0.125 > 0.5 오류<br>  -> 자연수 감각 전이: 125 > 5이므로 0.125 > 0.5로 착각<br>- 3.5 = 3.50 이해 못함: 0이 붙으면 다른 수라고 인식<br>- 소수점 아래 자릿수와 크기의 관계 혼동<br>- 소수와 분수의 관계 이해 부족 |
| **문항 설계 지침** | - 자릿값 판(Place Value Chart) 활용 문항<br>- 소수 크기 비교: "0.5와 0.125 중 어느 것이 큰가?"<br>- 동치 소수 확인: "3.5와 3.50은 같은가?"<br>- 소수와 분수 상호 변환 |

#### 세부 개념 02: 소수 덧뺄셈 계산 (`concept-e4-dec-op-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **소수점 위치 맞추어 세로 정렬**: 가장 중요한 원칙<br>- 자연수 덧뺄셈과 동일한 알고리즘, 소수점만 정렬<br>- 자릿수가 다른 경우: 빈자리에 0 채워 정렬<br>  (예: 2.5 + 1.23 -> 2.50 + 1.23)<br>- 결과에서 소수점 위치 확인 |
| **주요 오개념** | - **자릿수 정렬 오류** (가장 빈출): 2.5 + 12에서 5와 2를 더함<br>  -> 소수점이 아닌 끝자리를 맞추려 함<br>- 자릿수가 다른 소수 계산 시 0 채우기 누락<br>- 받아올림/받아내림 시 소수점 이동 오류<br>- 계산 후 소수점 찍는 것을 잊음 |
| **문항 설계 지침** | - 소수점 기준 정렬 강조 문항<br>- 자릿수가 다른 소수의 연산: 0 채우기 과정 포함<br>- 잘못된 정렬의 오류 찾기<br>- 어림으로 결과의 합리성 확인 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 소수 크기 비교 | 소수 길이와 크기 구분 | 하 |
| 자릿값 | 소수의 특정 자리 값 | 중 |
| 기본 덧뺄셈 | 같은 자릿수 소수 연산 | 중 |
| 자릿수 다른 연산 | 0 채워 정렬 후 계산 | 중상 |
| 오류 수정 | 잘못된 정렬/계산 찾기 | 상 |

### 10단원: 사각형 - 도형의 분류와 포함 관계

> 수직과 평행의 개념을 도입하고, 사각형을 체계적으로 분류하며 포함 관계를 이해하는 단원. 논리적 사고와 도형의 성질 탐구의 핵심.

#### 세부 개념 01: 수직과 평행 (`concept-e4-quad-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **수직**: 두 직선이 만나 직각(90도) 형성<br>- **평행**: 두 직선이 아무리 늘려도 만나지 않음<br>- **평행선 사이의 거리**: 한 직선에서 다른 직선에 수선을 내렸을 때의 길이<br>- 수직/평행의 기호: $\perp$ (수직), $\parallel$ (평행)<br>- 삼각자/각도기를 이용한 수직선/평행선 작도 |
| **주요 오개념** | - **평행선 사이 거리를 비스듬히 잼**: 수선(수직 거리)으로 재야 함<br>- 만나는 두 직선은 모두 수직이라고 착각: 직각이어야만 수직<br>- 평행선이 가까우면 "조금만 더 가면 만난다"고 오해<br>- 삼각자로 평행선 긋기에서 삼각자 이동 방향 혼동 |
| **문항 설계 지침** | - 수직인 직선과 수직이 아닌 직선 구분 문항<br>- 평행선 사이 거리 측정: 수선과 사선 구분<br>- 삼각자/각도기를 이용한 작도 문항<br>- 실생활에서 수직/평행 찾기 |

#### 세부 개념 02: 사각형 분류와 포함 관계 (`concept-e4-quad-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **분류 체계**:<br>  - 사다리꼴: 평행한 변 1쌍 이상<br>  - 평행사변형: 평행한 변 2쌍<br>  - 마름모: 네 변의 길이 모두 같음<br>  - 직사각형: 네 각이 모두 직각<br>  - 정사각형: 네 각 직각 + 네 변 같음<br>- **포함 관계**:<br>  정사각형 $\subset$ 직사각형 $\subset$ 평행사변형 $\subset$ 사다리꼴<br>  정사각형 $\subset$ 마름모 $\subset$ 평행사변형 $\subset$ 사다리꼴 |
| **주요 오개념** | - **포함 관계 이해 부족** (가장 빈출):<br>  - "정사각형은 직사각형인가?" -> 예 (네 각 직각이므로)<br>  - "정사각형은 마름모인가?" -> 예 (네 변 같으므로)<br>  - "직사각형은 평행사변형인가?" -> 예 (평행 2쌍이므로)<br>- 마름모를 다이아몬드 형태로만 인식: 회전된 정사각형도 마름모<br>- 사다리꼴의 정의에서 "이상" 해석 오류: 평행한 변 2쌍이면 사다리꼴 아니라고 착각 |
| **문항 설계 지침** | - 벤다이어그램으로 포함 관계 평가<br>- "평행사변형을 모두 고르시오"에 직사각형/정사각형 포함<br>- 다양한 방향의 마름모 제시: 정사각형 모양 마름모 포함<br>- 사각형 분류 서술형: "왜 정사각형은 직사각형인가?" |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 수직/평행 식별 | 주어진 도형에서 수직/평행 찾기 | 하 |
| 사각형 이름 | 조건 보고 사각형 종류 답하기 | 중 |
| 포함 관계 | "정사각형은 직사각형인가?" | 중상 |
| 벤다이어그램 | 사각형 분류를 벤다이어그램으로 표현 | 상 |
| 모두 고르기 | "평행사변형을 모두 고르시오" | 상 |

### 11단원: 꺾은선그래프 - 변화와 추세의 시각화

> 연속적 변량(시간에 따른 변화)을 시각화하고 추세를 파악하는 단원. 막대그래프와의 차이를 이해하고, 적절한 그래프를 선택하는 능력 배양.

#### 세부 개념 01: 읽기와 해석 (`concept-e4-line-graph-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **꺾은선그래프의 특성**: 연속적 변량(시간에 따른 변화) 시각화에 적합<br>- **선분 방향**: 올라가면 증가, 내려가면 감소<br>- **기울기**: 급하면 변화 큼, 완만하면 변화 작음<br>- **막대그래프와 비교**: 항목별 비교 -> 막대그래프, 변화 추이 -> 꺾은선그래프<br>- 추세 파악: 전체적인 증가/감소/유지 경향 읽기 |
| **주요 오개념** | - **항목별 비교에도 꺾은선그래프 사용**: 범주형 자료에는 막대그래프가 적합<br>- 기울기와 변화량의 관계 이해 부족: "급한 것 = 큰 변화" 연결 못함<br>- 점과 점 사이 구간의 의미 해석 못함<br>- 미래 예측 시 추세 반영 못함 |
| **문항 설계 지침** | - 자료 성격에 따른 그래프 선택 문제: "이 자료에 적절한 그래프는?"<br>- 기울기를 통한 변화 크기 비교<br>- 추세 기반 합리적 추론: "다음 달은 어떨 것으로 예상되는가?"<br>- 꺾은선그래프 해석 서술형 |

#### 세부 개념 02: 그리기와 물결선 (`concept-e4-line-graph-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **물결선($\sim$)**: 불필요한 구간 생략하여 변화를 더 잘 보이게 함<br>- **눈금 설정**: 자료 범위에 맞는 적절한 눈금 간격 결정<br>- **그래프 작성 절차**: 축 설정 -> 눈금/물결선 결정 -> 점 찍기 -> 선분 연결 -> 제목<br>- 물결선 사용 시 주의: 변화가 과장되어 보일 수 있음 |
| **주요 오개념** | - 물결선의 의미와 역할 이해 부족: 왜 생략하는지 모름<br>- 물결선 사용 시 눈금 시작점 오해: 0에서 시작하지 않으면 혼란<br>- 눈금 간격 부적절하게 설정<br>- 점을 찍고 선분으로 연결하는 순서 혼동 |
| **문항 설계 지침** | - 물결선이 있는/없는 그래프 비교: 같은 자료의 다른 인상<br>- 주어진 자료를 꺾은선그래프로 그리기<br>- 적절한 눈금과 물결선 결정 문항<br>- 물결선 사용의 장단점 서술형 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 값 읽기 | 꺾은선그래프에서 값 읽기 | 하 |
| 변화 해석 | 증가/감소/변화 크기 파악 | 중 |
| 그래프 선택 | 자료에 적절한 그래프 종류 | 중 |
| 물결선 이해 | 물결선의 역할과 효과 | 중상 |
| 그래프 작성 | 자료를 꺾은선그래프로 그리기 | 상 |

### 12단원: 다각형 - 도형의 일반화

> 다각형의 정의와 대각선, 정다각형의 조건을 학습하여 도형 인식을 일반화하는 단원. 논리적 분류와 필요충분조건 이해의 기초.

#### 세부 개념 01: 다각형 정의와 대각선 (`concept-e4-polygon-01`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **다각형**: 선분으로만 둘러싸인 닫힌 도형<br>- **꼭짓점과 변의 수**: n각형은 꼭짓점 n개, 변 n개<br>- **대각선**: 이웃하지 않는 두 꼭짓점을 연결한 선분<br>- **대각선 수**: 삼각형 0개, 사각형 2개, 오각형 5개...<br>- 볼록 다각형과 오목 다각형 구분 |
| **주요 오개념** | - **삼각형에 대각선이 없다는 사실 간과**: 모든 꼭짓점이 이웃이므로 대각선 0개<br>- 오목 다각형을 다각형이 아니라고 판단<br>- 곡선이 포함된 도형을 다각형이라 착각: 선분으로만 이루어져야 함<br>- 대각선 수 세기에서 중복 계산 |
| **문항 설계 지침** | - 다각형 판별: 다각형인 것과 아닌 것 구분 (곡선 포함, 열린 도형 등)<br>- 대각선 수 세기: 체계적 나열 방법 유도<br>- "삼각형에서 대각선을 그을 수 있는가?"<br>- 대각선으로 나누어지는 삼각형 수 탐구 |

#### 세부 개념 02: 정다각형의 조건 (`concept-e4-polygon-02`)

| 구분 | 내용 |
|------|------|
| **핵심 개념** | - **정다각형의 정의**: 변의 길이가 모두 같음 **그리고** 각의 크기가 모두 같음 (두 조건 **모두** 필수)<br>- **반례를 통한 이해**:<br>  - 마름모: 변만 같음 -> 정다각형 아님<br>  - 직사각형: 각만 같음 -> 정다각형 아님<br>  - 정사각형: 변도 같고 각도 같음 -> 정다각형!<br>- 정삼각형, 정사각형, 정오각형, 정육각형 등 |
| **주요 오개념** | - **변만 같으면 정다각형** (가장 빈출): 마름모가 반례<br>- **각만 같으면 정다각형**: 직사각형이 반례<br>- 두 조건의 필요충분성 이해 부족<br>- 정다각형의 대칭성(대칭축 수, 회전 대칭) 인식 부족 |
| **문항 설계 지침** | - 마름모 제시 후 "정다각형인가?" 질문 (반례 활용)<br>- 직사각형 제시 후 "정다각형인가?" 질문<br>- 두 조건의 필요충분성 확인: "변만 같으면 충분한가?"<br>- 정다각형의 성질 탐구: 대칭축, 내각의 크기 |

| 문항 유형 | 설명 | 난이도 |
|-----------|------|--------|
| 다각형 판별 | 다각형인 것과 아닌 것 구분 | 하 |
| 대각선 수 | 다각형의 대각선 수 세기 | 중 |
| 정다각형 판별 | 주어진 도형이 정다각형인지 | 중상 |
| 반례 활용 | 마름모/직사각형으로 조건 확인 | 상 |
| 필요충분조건 | 두 조건 모두 필요함을 설명 | 상 |

## 문제 개발 전략

### 1. 오개념 격파 문항 설계

| 나쁜 예 | 좋은 예 |
|---------|---------|
| "$\frac{1}{3} + \frac{1}{3}$을 계산하시오." | "$\frac{1}{3} + \frac{1}{3} = \frac{2}{6}$이 틀린 이유를 그림으로 설명하시오." |
| "이 사각형의 이름을 쓰시오." | "정사각형은 직사각형이라고 할 수 있는가? 이유를 설명하시오." |
| "삼각형을 분류하시오." | "둔각삼각형에도 예각이 있는가? 있다면 왜 예각삼각형이 아닌가?" |

### 2. 반례를 통한 개념 심화

| 오개념 | 반례 | 교정 도구 |
|--------|------|-----------|
| 변이 길면 각도 크다 | 변 길이 다르나 각 같은 도형 쌍 | 각도기 측정 활동 |
| 변만 같으면 정다각형 | 마름모 (변 같지만 각 다름) | 조건 비교표 |
| 각만 같으면 정다각형 | 직사각형 (각 같지만 변 다름) | 조건 비교표 |
| 예각 있으면 예각삼각형 | 둔각삼각형의 예각 2개 | 세 각 측정 후 분류 |
| 정사각형 != 직사각형 | 정사각형의 네 각이 모두 직각 | 벤다이어그램 |
| 분모끼리 더하기 | 파이 조각 $\frac{1}{3}+\frac{1}{3}$은 $\frac{2}{3}$이지 $\frac{2}{6}$이 아님 | 영역 모델 |

### 3. 분수/소수 오개념 교정 핵심

| 오개념 | 교정 도구 |
|--------|-----------|
| $\frac{1}{3}+\frac{1}{3}=\frac{2}{6}$ (분모끼리 더함) | 파이 조각 그림, "단위(분모) 통일" 비유 |
| 받아내림 시 10 더함 | 단계별 과정 제시 -> 오류 찾기 |
| 소수가 길면 크다 (0.125 > 0.5) | 자릿값 판(Place Value Chart), 수직선 |
| 소수점 정렬 오류 | 세로셈 정렬 연습, 0 채우기 강조 |

### 4. 도형 분류와 포함 관계 지도

| 관계 | 설명 | 출제 포인트 |
|------|------|-------------|
| 정삼각형 $\subset$ 이등변삼각형 | 세 변 같음 -> 당연히 두 변도 같음 | "정삼각형은 이등변삼각형인가?" |
| 정사각형 $\subset$ 직사각형 | 네 각 직각 + 네 변 같음 -> 네 각 직각 | "정사각형은 직사각형인가?" |
| 정사각형 $\subset$ 마름모 | 네 각 직각 + 네 변 같음 -> 네 변 같음 | "정사각형은 마름모인가?" |
| 직사각형 $\subset$ 평행사변형 | 네 각 직각 -> 대변 평행 | "직사각형은 평행사변형인가?" |

> **포함 관계 지도의 핵심**: "특수한 것은 일반적인 것에 포함된다"

### 5. 역량 기반 문항 설계

| 핵심 역량 | 문항 설계 포인트 | 예시 |
|-----------|------------------|------|
| **문제해결** | 정형화되지 않은 문제 | 나머지의 맥락적 해석 (올림/버림 판단) |
| **추론** | 논리적 정당화 | "왜 삼각형 내각의 합은 항상 180도인가?" |
| **의사소통** | 표현 변환 (말-식-그림) | 분수 연산을 그림과 식으로 동시 표현 |
| **연결** | 단원 간 연결 | 각도 -> 삼각형 분류, 수직/평행 -> 사각형 분류 |
| **정보처리** | 데이터 해석 | 막대그래프/꺾은선그래프 해석 및 적절한 선택 |

### 7. 그래프 지도 전략

- **막대그래프 vs 꺾은선그래프 비교**:

- **물결선의 올바른 사용**:
  - 필요한 경우에만 사용
  - 변화가 과장될 수 있음을 인식
- **눈금 설정의 중요성**:
  - 자료 범위에 맞는 적절한 간격
