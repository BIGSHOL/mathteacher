# Design System (기초 디자인 시스템)

> 비기술자도 이해할 수 있는 디자인 언어로 작성합니다.
> 듀오링고 스타일의 게임화 UI를 목표로 합니다.
> 최소 접근성 패키지를 포함합니다.

---

## MVP 캡슐

| # | 항목 | 내용 |
|---|------|------|
| 1 | 목표 | 강사 효율 극대화 + 학생 자기주도 학습 환경 구축 |
| 2 | 페르소나 | 초등~고1 학생 (공통수학), 학원 강사/관리자 |
| 3 | 핵심 기능 | FEAT-1: 개념 테스트 (수학 개념 이해도 확인) |
| 4 | 성공 지표 (노스스타) | 강사 개별 확인 시간 50% 절감 |
| 5 | 입력 지표 | 주간 테스트 완료 학생 비율, 평균 정답률 향상 |
| 6 | 비기능 요구 | 응답 시간 < 500ms, 모바일/태블릿 반응형 지원 |
| 7 | Out-of-scope | AI 개인 맞춤 추천, 학부모 앱, 결제 시스템 |
| 8 | Top 리스크 | 학생들이 재미없어서 안 쓸 수 있음 |
| 9 | 완화/실험 | 듀오링고 스타일 게임화 요소 적용 |
| 10 | 다음 단계 | 개념 테스트 문제 DB 설계 |

## 1. 디자인 철학

### 1.1 핵심 가치

| 가치 | 설명 | 구현 방법 |
|------|------|----------|
| **재미 (Fun)** | 학습이 게임처럼 느껴지도록 | 애니메이션, 피드백, 보상 시스템 |
| **속도감 (Speed)** | 빠른 응답, 즉각적인 피드백 | 200ms 이내 채점 응답, 부드러운 전환 |
| **격려 (Encouragement)** | 실패해도 다시 도전하고 싶게 | 긍정적 메시지, 진행 상황 시각화 |
| **단순함 (Simplicity)** | 초등학생도 바로 사용 가능 | 한 화면에 한 가지 액션, 큰 버튼 |

### 1.2 참고 서비스 (무드보드)

| 서비스 | 참고할 점 | 참고하지 않을 점 |
|--------|----------|-----------------|
| **듀오링고** | 게임화 (레벨, 스트릭, 콤보), 귀여운 피드백 | 캐릭터 일러스트 (개발 비용) |
| **칸아카데미** | 깔끔한 문제 화면, 진행 바 | 텍스트 중심 UI |
| **마스매티카 (매쓰매티카)** | 수학 특화 UI, 수식 표현 | 복잡한 그래프 |

## 3. 타이포그래피

### 3.1 폰트 패밀리

| 용도 | 폰트 | 대안 |
|------|------|------|
| 본문/UI | Pretendard | system-ui, -apple-system, sans-serif |
| 숫자/수식 | Roboto Mono | monospace |
| 수학 기호 | KaTeX (렌더링) | - |

### 3.2 타입 스케일

| 이름 | 크기 | 굵기 | 용도 |
|------|------|------|------|
| Display | 36px | Bold (700) | 레벨업!, 축하 메시지 |
| H1 | 28px | Bold (700) | 테스트 제목 |
| H2 | 22px | SemiBold (600) | 섹션 제목 |
| H3 | 18px | SemiBold (600) | 카드 제목, 문제 번호 |
| Body Large | 18px | Regular (400) | 문제 본문 (큰 글씨) |
| Body | 16px | Regular (400) | 일반 본문 |
| Body Small | 14px | Regular (400) | 선택지, 설명 |
| Caption | 12px | Regular (400) | 힌트, 부가 정보 |

### 3.3 수학 표현

- **분수**: `1/2` 또는 KaTeX `\frac{1}{2}`
- **지수**: `x²` 또는 KaTeX `x^2`
- **특수 기호**: KaTeX 렌더링

## 6. 게임화 애니메이션

### 6.1 정답 피드백

```
1. 화면 중앙에 "정답!" 텍스트 팝업 (0.3초)
2. 초록색 원형 파티클 퍼짐 (0.5초)
3. 점수 +10 날아오르는 텍스트 (0.4초)
4. 콤보 카운터 증가 (0.2초)
```

### 6.2 오답 피드백

```
1. 선택지 흔들림 애니메이션 (0.3초)
2. "아쉬워요~" 텍스트 페이드인 (0.3초)
3. 정답 표시 + 해설 슬라이드업 (0.3초)
```

### 6.3 레벨업

```
1. 화면 전체 골드 플래시 (0.2초)
2. "레벨 UP!" 대형 텍스트 + 별 파티클 (1초)
3. 새 레벨 배지 표시 (0.5초)
4. 축하 사운드 (옵션)
```

### 6.4 콤보 애니메이션

| 콤보 | 애니메이션 |
|------|-----------|
| x3 | 콤보 배지 바운스 |
| x5 | 화면 테두리 주황색 글로우 |
| x10 | 배경 퍼플 그라데이션 + "무적모드!" |

## 7. 반응형 디자인

### 7.1 브레이크포인트

| 이름 | 너비 | 대상 |
|------|------|------|
| Mobile | < 640px | 스마트폰 |
| Tablet | 640px ~ 1024px | 태블릿, 작은 노트북 |
| Desktop | > 1024px | PC, 대형 태블릿 |

### 7.2 레이아웃 변화

**문제 화면:**
| 화면 | 레이아웃 |
|------|----------|
| Mobile | 문제 + 선택지 수직 배열, 전체 너비 |
| Tablet | 문제 + 선택지 수직 배열, 최대 600px |
| Desktop | 문제 + 선택지 수직 배열, 최대 800px, 좌우 여백 |

**강사 대시보드:**
| 화면 | 레이아웃 |
|------|----------|
| Mobile | 사이드바 숨김, 햄버거 메뉴 |
| Tablet | 사이드바 아이콘만 |
| Desktop | 사이드바 전체 + 컨텐츠 |

## 8. 접근성 체크리스트

### 8.1 필수 (MVP)

- [x] **색상 대비**: 텍스트와 배경의 대비율 4.5:1 이상 (WCAG AA)
- [x] **포커스 링**: 키보드 탐색 시 포커스 표시 명확 (3px 민트 링)
- [x] **클릭 영역**: 버튼/선택지 최소 48x48px (모바일 터치)
- [x] **에러 표시**: 색상 + 아이콘 + 텍스트 병행 (색맹 고려)
- [x] **폰트 크기**: 문제 본문 최소 16px, 선택지 14px
- [x] **터치 타겟 간격**: 최소 8px 간격

### 8.2 권장 (v2)

- [ ] 키보드 전체 탐색 가능 (Tab, Enter, Arrow)
- [ ] 스크린 리더 호환 (ARIA 라벨)
- [ ] 애니메이션 줄이기 옵션 (prefers-reduced-motion)
- [ ] 고대비 모드
