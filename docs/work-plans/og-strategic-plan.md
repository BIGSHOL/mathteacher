# Math Test 프로젝트 OG (Objective & Goal) 전략 계획서

> **작성일**: 2026-02-03  
> **프로젝트**: Math Test - 수학 개념 및 연산 테스트 프로그램  
> **상태**: 전략 수립 중

---

## 1. 프로젝트 비전 (Vision)

> **"선생님의 손이 닿지 않는 시간에도 학생들이 스스로 성장하는 수학 학습 환경"**

학원 강사 1명이 30명의 학생을 개별 관리하는 비효율을 해소하고,  
학생들이 **자기주도적**으로 수학 실력을 진단·향상할 수 있는 디지털 학습 도구를 구축한다.

## 3. 핵심 지표 체계 (KPI Framework)

### 3.1 노스스타 지표 (North Star Metric)

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   📊 강사 개별 확인 시간 절감률: 50%                      │
│                                                         │
│   = (기존 확인 시간 - 현재 확인 시간) / 기존 확인 시간     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 입력 지표 (Leading Indicators)

| 영역 | 지표 | 목표 | 측정 방법 |
|------|------|------|-----------|
| **참여** | 주간 테스트 완료율 | ≥ 70% | 주 3회 이상 테스트 완료 학생 비율 |
| **학습** | 평균 정답률 향상 | +10%/월 | 월간 평균 정답률 상승폭 |
| **재방문** | DAU/MAU 비율 | ≥ 30% | 일간/월간 활성 사용자 비율 |
| **만족** | 게임화 만족도 | ≥ 4.0/5.0 | 학생 설문 |

### 3.3 품질 지표 (Quality Indicators)

| 영역 | 지표 | 목표 |
|------|------|------|
| 성능 | API 응답 시간 (P95) | < 500ms |
| 안정성 | 서비스 가용성 | ≥ 99% |
| 사용성 | 모바일 이탈률 | < 30% |

## 6. 기술 전략 (Technical Strategy)

### 6.1 아키텍처 원칙

| 원칙 | 설명 | 적용 |
|------|------|------|
| **LLM 불필요** | 연산/개념 문제는 템플릿 기반 생성 | 문제 생성 엔진 자체 개발 |
| **학년별 독립** | 각 학년 내 Lv.1~10 독립 운영 | 200개 난이도 슬롯 |
| **이원 트랙** | 연산 실력과 개념 실력 분리 측정 | category: computation / concept |
| **교육과정 기반** | 2022 개정 교육과정 반영 | 학년별 성취기준 매핑 |

### 6.2 기술 스택 확정

| 영역 | 기술 | 선택 근거 |
|------|------|-----------|
| **Backend** | FastAPI + PostgreSQL | 빠른 개발, 타입 안전성 |
| **Frontend** | React + Vite + TypeScript | 모던 개발 환경, 빠른 빌드 |
| **상태관리** | Zustand | 경량, 간단한 API |
| **스타일** | TailwindCSS | 빠른 UI 개발 |
| **테스트** | pytest + Vitest + Playwright | 백엔드/프론트/E2E 커버 |

## 7. 리스크 관리 (Risk Management)

### 7.1 Top 5 리스크 보드

| # | 리스크 | 확률 | 영향 | 완화 전략 | 트리거 | 대응 |
|---|--------|------|------|-----------|--------|------|
| R1 | 학생이 재미없어서 안 씀 | 중 | 높음 | 듀오링고 스타일 게임화 | 주간 완료율 < 30% | 보상 체계 강화 |
| R2 | 문제 난이도 부적절 | 중 | 중 | 10단계 세분화 + 적응형 | 정답률 > 95% or < 30% | 난이도 재조정 |
| R3 | 강사가 기존 방식 고수 | 중 | 중 | 간단한 대시보드 | 강사 접속 < 주 1회 | UX 개선, 교육 |
| R4 | 서버 지연으로 UX 저하 | 낮 | 중 | P95 < 500ms 목표 | P95 > 1초 | 캐싱, 인프라 확장 |
| R5 | 모바일 사용성 문제 | 중 | 중 | 모바일 퍼스트 설계 | 모바일 이탈률 > 50% | UI 최적화 |

### 7.2 가정 검증 계획

| 가정 | 검증 방법 | 검증 시기 |
|------|-----------|-----------|
| 게임화가 동기부여에 효과적 | MVP 파일럿 (5명, 2주) | 3월 |
| 강사가 대시보드 활용 | 강사 접속 로그 + 인터뷰 | 4월 |
| 학부모가 집에서 숙제로 활용 | 가정 접속 비율 추적 | 5월 |
