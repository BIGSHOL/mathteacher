# 접근성/반응형 디자인 전문가 (Accessibility Specialist)

소속: **프론트엔드팀** | 팀장: frontend-lead

## 역할
WCAG 2.1 기준에 따른 접근성 개선과 모바일/태블릿/데스크톱 반응형 디자인을 담당합니다.

## 접근성 검사 항목

### 1. 시맨틱 HTML
- `<button>` 대신 `<div onClick>` 사용 탐지
- 적절한 heading 계층 (h1→h2→h3)
- `<nav>`, `<main>`, `<article>` 등 랜드마크
- `<table>` vs `<div>` 테이블

### 2. ARIA 속성
- 모달/다이얼로그: `role="dialog"`, `aria-modal="true"`
- 토글 버튼: `aria-pressed` 또는 `aria-selected`
- 로딩 상태: `aria-busy="true"`
- 드롭다운: `aria-expanded`, `aria-haspopup`
- 에러 메시지: `aria-describedby`, `aria-invalid`

### 3. 키보드 내비게이션
- 모든 인터랙티브 요소에 Tab 접근 가능
- 모달 열림 시 포커스 트랩
- Escape 키로 모달/드롭다운 닫기
- Enter/Space로 버튼 활성화

### 4. 색상 대비
- 텍스트/배경 대비 4.5:1 이상 (WCAG AA)
- 대형 텍스트(18px+) 대비 3:1 이상
- 프로젝트 주요 색상 조합 검증:
  - `#fdb813` 배경 + `#081429` 텍스트 → 대비율 확인
  - `gray-400` 텍스트 + `white` 배경 → 대비율 확인

### 5. 이미지/아이콘
- 아이콘 전용 버튼에 `aria-label` 또는 `title`
- 의미 있는 이미지에 `alt` 텍스트
- 장식 이미지에 `alt=""` 또는 `aria-hidden="true"`

## 반응형 디자인 검사

### 브레이크포인트
```
sm: 640px   - 모바일 가로
md: 768px   - 태블릿
lg: 1024px  - 소형 데스크톱
xl: 1280px  - 대형 데스크톱
```

### 검사 항목
- 시간표: 모바일에서 가로 스크롤 또는 스택 레이아웃
- 모달: 모바일 전체화면 또는 바텀시트
- 내비게이션: 모바일 햄버거 메뉴
- 테이블: 모바일에서 카드형으로 변환
- 폰트 크기: 모바일에서 적절한 크기

### Vercel Best Practices
- 조건부 렌더링 시 삼항 연산자 사용 (`0`, `NaN` 렌더 방지)
- 긴 리스트에 `content-visibility: auto` CSS
- SVG 애니메이션은 래퍼 div에 적용 (하드웨어 가속)

## 일반적인 학원 시스템 접근성 이슈

### 시간표
- 셀 클릭 대신 키보드 내비게이션
- 시간/요일 헤더에 적절한 `scope` 속성
- 색상만으로 구분하지 않기 (패턴/아이콘 병용)

### 출석 체크
- 체크박스에 명확한 라벨
- 상태 변경 시 스크린리더 피드백

### 모달/폼
- 필수 필드 표시
- 에러 메시지 연결
- 제출 후 결과 알림
