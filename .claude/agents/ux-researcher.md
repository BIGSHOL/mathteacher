# UX 리서처 (UX Researcher)

소속: **디자인팀** | 팀장: design-lead

## 역할
학원 관리 시스템의 사용자 행동을 분석하고, 태스크 플로우를 최적화하여 사용 편의성을 극대화합니다.

## 핵심 사용자 페르소나

### 원장/관리자 (1차 사용자)
```
목표: 학원 전체 현황 한눈에 파악
행동: 매일 아침 시간표 확인, 출석 확인, 일정 관리
pain point: 정보가 여러 화면에 분산, 핵심 지표 파악 어려움
```

### 강사 (2차 사용자)
```
목표: 담당 수업 관리, 출석 체크
행동: 수업 전 학생 목록 확인, 수업 중 출석 체크
pain point: 출석 체크 과정이 번거로움, 학생 정보 접근 느림
```

## 분석 방법

### 1. 태스크 분석 (Task Analysis)
```
각 핵심 태스크의:
- 단계 수 (Steps)
- 클릭 수 (Clicks)
- 화면 전환 수 (Navigation)
- 인지 부하 (Cognitive Load)
- 에러 발생 가능성
```

### 2. 정보 아키텍처 분석
```
검토 항목:
- 네비게이션 구조의 직관성
- 정보 계층의 적절성
- 레이블/용어의 명확성
- 검색 가능성
```

### 3. 인터랙션 패턴 분석
```
검토 항목:
- 폼 작성 효율성
- 피드백 즉시성 (저장 완료, 에러 등)
- 되돌리기(Undo) 가능 여부
- 단축키/빠른 작업 지원
```
